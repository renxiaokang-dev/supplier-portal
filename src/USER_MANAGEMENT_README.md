# 用户账户管理和登录系统

这是一个完整的用户账户管理和登录系统，与供应商报价单管理模块集成，采用深蓝色主题（#0b5698）。

## 功能特性

### 登录功能

#### 1. 密码登录
- 支持使用用户名/邮箱/手机号登录
- 密码显示/隐藏切换功能
- 完整的表单验证和错误提示
- 隐私政策勾选确认

#### 2. 验证码登录
- **邮箱验证码登录**：输入邮箱地址接收验证码
- **短信验证码登录**：输入手机号接收验证码
- 60秒倒计时防止重复发送
- 自动格式化输入（仅允许数字）

#### 3. 首次登录
- 仅显示验证码登录选项
- 登录后自动跳转到账户设置页面

### 账户管理

#### 1. 账户设置（首次登录）
- 设置用户名（3-20个字符，支持字母、数字、下划线、中文）
- 设置密码（至少8个字符，必须包含数字和字母）
- 实时密码强度检测（弱/中/强）
- 密码要求检查列表
- 确认密码匹配验证

#### 2. 修改密码
- 输入当前密码验证
- 设置新密码
- 实时密码强度指示器
- 密码要求检查
- 确认新密码

#### 3. 用户菜单
- 点击顶部用户名触发下拉菜单
- "修改密码"选项
- "退出登录"选项

## 页面路由

- `/demo` - 演示页面（展示所有功能）
- `/login` - 密码登录 / 验证码登录
- `/first-login` - 首次登录（仅验证码登录）
- `/account-setup` - 账户设置页面
- `/change-password` - 修改密码页面
- `/` - 主应用（需要登录）

## 使用方法

### 访问演示页面

1. 访问 `/demo` 页面查看所有功能的入口
2. 点击相应按钮体验不同的登录流程

### 密码登录流程

1. 访问 `/login`
2. 选择"密码登录"标签页
3. 输入用户名/邮箱/手机号和密码
4. 勾选隐私政策
5. 点击"登录"按钮

### 验证码登录流程

1. 访问 `/login`
2. 选择"验证码登录"标签页
3. 选择"邮箱"或"短信"子标签页
4. 输入邮箱地址或手机号
5. 点击"发送验证码"按钮
6. 输入收到的6位验证码
7. 勾选隐私政策
8. 点击"登录"按钮

### 首次登录流程

1. 访问 `/first-login`
2. 使用验证码登录
3. 自动跳转到账户设置页面
4. 设置用户名和密码
5. 完成设置后进入主应用

### 修改密码

1. 登录后点击顶部用户名
2. 选择"修改密码"
3. 输入当前密码
4. 设置新密码
5. 确认新密码
6. 点击"确认修改"

## 表单验证规则

### 用户名
- 长度：3-20个字符
- 允许：字母、数字、下划线、中文

### 密码（登录）
- 最少6个字符

### 新密码（设置/修改）
- 最少8个字符
- 必须包含数字
- 必须包含字母
- 推荐包含大写字母
- 推荐包含特殊字符

### 邮箱
- 标准邮箱格式验证

### 手机号
- 11位数字
- 中国大陆手机号格式

### 验证码
- 6位数字

## 密码强度检测

系统会根据以下标准实时检测密码强度：

- **弱（红色）**：基本要求未满足
- **中等（橙色）**：满足基本要求
- **强（绿色）**：包含大小写字母、数字和特殊字符

评分标准：
- 长度 ≥ 8个字符：+1分
- 长度 ≥ 12个字符：+1分
- 包含数字：+1分
- 包含小写字母：+1分
- 包含大写字母：+1分
- 包含特殊字符：+1分

## 设计规范

### 主题色
- 主色调：`#0b5698` (深蓝色)
- 悬停色：`#0d679f`
- 渐变背景：`from-[#0b5698] to-[#0d679f]`

### 状态颜色
- 成功/强密码：`#10b981` (绿色)

- 错误/弱密码：`#ef4444` (红色)

### 组件样式
- 圆角：`rounded-lg`
- 阴影：`shadow-2xl`
- 卡片背景：白色
- 输入框高度：标准

## 技术实现

### 核心技术
- React 18
- React Router 6
- TypeScript
- Tailwind CSS v4
- Lucide React (图标)

### 状态管理
- AuthContext：用户认证状态
- QuotationContext：报价单数据（原有）
- LocalStorage：持久化登录状态

### 保护路由
- ProtectedRoute 组件：自动重定向未登录用户
- 登录状态持久化

### 表单处理
- 完整的客户端验证
- 实时错误提示
- 禁用状态管理
- 加载状态反馈

## 模拟功能

当前系统使用模拟数据进行演示：

1. **验证码发送**：模拟60秒倒计时，实际不发送验证码
2. **登录验证**：任何有效格式的输入都会通过验证
3. **数据持久化**：使用 LocalStorage 存储用户信息
